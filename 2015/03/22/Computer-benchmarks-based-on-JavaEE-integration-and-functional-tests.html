2. Projects

restart your machine before each project

2.1. Intrabundle project

integration tests using JBoss forge

2.1.1. checkout sources

  1. git clone https://github.com/rmpestano/intrabundle.git

  2. cd intrabundle

  3. git reset --hard 00f05bd2d8e9f600850eb628141716bc2821160d

2.1.2. download maven dependencies

mvn dependency:tree -Ptests

2.2. Execute build and tests

mvn clean test -Ptests

2.2.1. Execute build and tests using Jacoco agent

mvn clean test -P tests -Pjacoco

2.3. cdi-crud project

Java EE project with various kinds of Arquillian tests (it, ft, bdd)

2.3.1. checkout sources

  1. git clone https://github.com/rmpestano/cdi-crud.git

  2. cd cdi-crud

  3. git reset --hard 6f86c59bd619af8f7ce4997a3dc60786884911f4

2.3.2. Download dependencies

mvn dependency:tree -P wildfly-managed

2.3.3. Execute build and tests

mvn clean package -P tests -P wildfly-managed -Dversion.selenium=2.45.0

you may need to upgrade selenium version to run ft-tests (if browser opens when nothing happens), use build param -Dversion.selenium=2.45.0 for exemple.

2.3.4. Execute build and tests with jacoco agent

mvn clean package -P tests -P wildfly-managed -Pjacoco -Dversion.selenium=2.45.0

2.4. conventions archetype project

A bit more complex Java EE project with various kinds of Arquillian tests (it, ft, bdd)

2.4.1. checkout sources

  1. git clone https://github.com/conventions/archetype.git

  2. cd archetype

  3. git reset --hard 952a7b5f4ac58726f78c63034a9bcf8b1630b495

2.4.2. Download dependencies

mvn dependency:tree -P wildfly-managed

2.4.3. Execute build and tests

mvn clean package -P all-tests -P wildfly-managed -Dversion.selenium=2.45.0

2.4.4. Execute build and tests with jacoco agent

mvn clean package -P all-tests -P wildfly-managed -Pjacoco -Dversion.selenium=2.45.0
